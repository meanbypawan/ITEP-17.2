FastAPI Features
High performance (built on ASGI and Starlette)
Supports asynchronous programming (async/await)
Automatic request and response validation using Pydantic
Automatic Swagger UI (/docs) and ReDoc (/redoc)
Type hint support for validation and documentation
Built-in Dependency Injection system
Easy JSON serialization/deserialization
WebSocket support
Background task support
Security features (OAuth2, JWT, API Keys, etc.)
Easy to learn and developer-friendly
Excellent support for REST APIs and microservices
----------------------------------------------------------------------------
FastAPI Limitations
No built-in ORM (requires SQLAlchemy, SQLModel, etc.)
No built-in admin panel like Django
Smaller ecosystem compared to Django
Async programming can be difficult for beginners
Requires additional libraries for many common features
Less suitable for traditional server-rendered websites
No strict project structure/conventions for large projects
Debugging async code can be challenging
Benefits of async are reduced if using synchronous libraries
Not ideal for CMS or content-heavy web applications

Internal Working Flow of FastAPI Application:-
  Client Send the request
     |
  Uvicorn[Uvicorn Server Receive the request]
     |
  FastAPI Application
     |
  Starlette Framework
     |
   AnyIO
     |----Event Loop{async def} managed by uvicorn
     |----Worker Thread{def}  managed by AnyIO

Responsibility of each layer :-
1. Uvicorn :-
   a. It is responsible to start server
   b. Listening on port
   c. Accepting HTTP Request
   d. Creating the ASGI Scope
   e. Running event Loop
   
2. FastAPI :-
   a. It is responsible for route registration
   b. Dependency Injection (Depends)
   c. Request validation using pydantic
   d. Response Serialization
   e. OpenAPI/Swagger documentation
   f. Security Helper
   
3. Starlette :-
   a. It is responsible for Request & Response Object
   b. Handling Middleware
   c. Routing implementation
   d. Serving Static files
   e. Managing Jinja2 templatess
   f. Background Task
   g. Deciding how to execute task
4. AnyIO :- AnyIO is concurrency library. It is not responsible for route handing and Http Request handling. It provide
   thread pool to manage synchronous operation and default thread pool size is 40
